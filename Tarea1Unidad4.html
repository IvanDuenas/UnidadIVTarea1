Mecanismos de seguridad en formularios

Los formularios son una de las mayores fuentes de problemas de seguridad y spam en sitios y aplicaciones web. Algunos de estos mecanismos son:

Captchas

Una de las mayores problematicas de los formularios es que los robots se dediquen a enviar información, buscando o explotando vulnerabilidades detectadas. Es por ello que nos aseguraremos de que son realmente los que están interactuando con nuestros formularios. Para ello se puede incorporar un sistema de captcha adecuado. Existen diversas alternativas en el mercado, pero igual que aparecen desaparecen. Una que perdura y en la que podemos confiar es reCaptcha, un servicio ofrecido por Google. Básicamente es un sistema muy amistoso para el usuario que muchas veces se valida simplemente pulsando un checkbox. Seguro que hemos visto decenas de veces la interfaz de reCaptcha en los sitios web. Tiene un aspecto como este:

Evitar ataques CSRF

Otro de los males de los formularios son los ataques CSRF (del inglés Cross-site request forgery), que se producen al realizarse solicitudes POST desde otros dominios que los ciberdelincuentes aprovechan para apoderarse en sesiones abiertas sobre los sitios atacados. CSRF se basa generalmente en un token, que simplemente se valida antes de aceptar el procesamiento de la información que viene mediante solicitudes POST. La mayoria de los frameworks incluyen por defecto protección anti CSRF, una razón más para usarlos.

Protocolo HTTPS

Finalmente, pero no menos importante, está el protoclo HTTPS. Se podria considerar como un escudo para proteger los datos confidenciales del usuario y del servidor en general. Al activar HTTPS, se está asegurando que los mensajes de nuestros usuarios estén encriptados y lejos de delincuentes digitales.

Introducción a AJAX

Se refiere a un grupo de tecnologías que se utilizan para desarrollar aplicaciones web. Al combinar estas tecnologías, las páginas web parece que son más receptivas puesto que los paquetes pequeños de datos se intercambian con el servidor y las páginas web no se vuelven a cargar cada vez que un usuario realiza un cambio de entrada. Ajax permite que un usuario de la aplicación web interactúe con una página web sin la interrupción que implica volver a cagar la página web. La interacción del siito web ocurre rápidamente sólo con partes de la página de recarga y renovación.

Ajax se compone de las siguientes tecnologias:

  • XHTML y CSS para presentar información
  • DOM para visualizar e interactuar de forma dinámica la información presentada
  • El objeto XMLHttpRequest para manipular los datos de forma asíncrona con el servidor web
  • Se visualiza JavaScript para enlazar solicitudes e información de datos

    • En una aplicación web tradicional, las solicitudes HTTP, que se inician mediante la interacción del usuario con la intefaz web, se realizan a un servidor web. El servidor web procesa la solicitud y devuelve una página HTML al cliente. Durante el transporte HTTP, el usuario no puede interactuar con la aplicación web.

    En una aplicación web Ajax, no se interrumpe el usuario en interacciones con la aplicación web. El motor de Ajax o el intérprete JavaScript permite que el usuario interactúe con la aplicación web independientemente del transporte HTTP procedente del servidor o que tenga el servidor como destino representando la interfaz y gestionando las comunicaciones con el servidor en nombre del usuario.

    Limitaciones de Ajax

    Aunque Ajax es una técnica de desarrollo de aplicaciones web que se ha diseñado para que las páginas web sean más receptivas e interactivas con un usuario, Ajax presenta algunas limitaciones a tener en cuenta antes de desarrollar una aplicación basada en Ajax. Las limitaciones siguientes representan algunas de las desventajas principales:

  • Soporte de navegador: No todos los navegadores admiten JavaScript o el objeto XMLHttpRequest. Incluso entre navegadores que no ofrecen soporte a JavaScript y XMLHttpRequest, estos objetos se puede tratar de forma diferente. Se deben tener en cuenta todas las implementaciones del navegador de Ajax
  • Seguridad y privacidad de usuario: No se resuelven todos los puntos de vista. Es necesario tener en cuenta los problemas relacionados con la seguridad y la privacidad de usuario a la hora de desarrollar una aplicación Ajax.
  • Accesibilidad: Puesto que no todos los navegadores disponen de soporte para JavaScript o el objeto XMLHttpRequest, debe asegurarse de proporcionar una manera de hacer que la aplicación web sea accesible para todos los usuarios.
  • Motor de búsqueda: No es posible realizar búsquedas en aplicaciones Ajax; sin embargo, sí se pueden utilizar las caracteristicas y elementos de Ajax de una aplicación en la que se puedan realizar búsquedas.
    • JQuery

    jQuery, creado en 2006, es una biblioteca de JavaScript rápida, pequeña y rica en funciones que se ha convertido en una de las bibliotecas JavaScript más populares del mundo, utilizada por más del 70% de los 10 millones de sitios web más populares.

    Entre sus características más destacadas podemos mencionar que facilita la escritura de JavaScript, abstrae muchas incompatibilidades entre navegadores y aporta una gran cantidad de utilidades listas para usar.

    Además, hace cosas como el recorrido y la manipulación de documentos HTML, el manejo de eventos, la animación y Ajax mucho más simple con una API fácil de usar que funciona a través de una multitud de navegadores.

    Para qué sirve jQuery

    El uso de jQuery es realmente extenso además de diverso, sin embargo, podríamos condensarlo a que sirve principalmente para simplificar el desarrollo de JavaScript en sitios web. Entre sus utilidades más destacadas podemos profundizar en las siguientes:

    La manipulación del DOM es una de las fortalezas principales de JQuery, ayq ue nos permite seleccionar elementos de forma flexible utilizando selectores CSS, ya sea por ID, clase, atributo, entre otros. Una vez seleccionados, se pueden crear elementos HTML desde cero, insertarlos, eliminarlos, copiarlos o reemplazarlos.

    El manejo de eventos es otra caracteristica fundamental de jQuery, ya que nos permite asociar y eliminar manejadores de eventos a selectores de forma sencilla, con compatibilidad con todos los eventos del navegador

    La delegación de eventos optimiza el rendimiento, mientras que el accionador manual de eventos brinda mayor control sobre su ejecución. jQuery también implementa eventos personalizados para extender su funcionalidad según se requiera

    Por último, jQuery incluye una variedad de utilidades como iteradores y métodos para arreglos, parcheo de datos JSON, XML y cadenas de parámetros, métodos para serialización/deserialización de formularios, y detección de características de navegadores. Estas funciones facilitan muchas tareas comunes en el desarrollo web.

    Mootools

    El framework MooTools es un conjunto compacto y modular de herramientas para desarrollaodres profesionales en JacaScript que se distingue por sus posibilidades de aplicación gracias a un elegante y bien documentado grupo de API y por estar orientado a objetos. MooTools pone disposición del desarrollador varias funciones y clases para facilitar la programación y el diseño de proyectos web modernos y simplificar en gran medida el proceso de trabajo.

    A diferencia del tradicional JavaScript, MooTools se fundamenta sobre un sistema de herencia de clases que sirve de base para la programación orientada a objetos y, por lo tanto, también para todos los componentes incluidos en el framework. Este sistema no solo se caracteriza por ser fácilmente escalable sino también por que permite escribir código fuente flexible y reutilizable.

    Prototype

    Se refiere a un mecanismo en JavaScript que permite a los objetos compartir propiedades y métodos. Es un concepto fundamental del lenguaje JavaScript, relacionado con la herencia y el diseño de objetos. JacaScript usa un sistema de herencia basado en prototipos, lo que significa que los objetos pueden heredar directamente de otros objetos. En lugar de usar clases como en otros lenguajes de programación orientados a objetos, JacaScript tiene lo que se llama una cadena de prototipos, en la cual cada objeto puede tener un enlace a un objeto prototipo del que puede heredar propiedades y métodos.

    Este modelo de herencia es lo que diferencia a JavaScript de otros lenguajes orientados a objetos, y hace posible la reutilización eficiente del código sin necesidad de clases estrictas

    Introducción a AJAX

    Introducción a AJAX

    La siguiente imagen compara dos modelos de interacción entre el cliente y el servidor en aplicaciones web: el modelo clásico y el modelo Ajax.

    Modelo clásico de aplicación web (izquierda)

    Flujo de datos:

    • La interfaz de usuario del navegador cliente envía una petición HTTP al servidor.
    • El servidor procesa la solicitud y devuelve una respuesta completa recargando toda la página.

    Componentes del servidor:

    • Servidor web: Procesa las solicitudes HTTP.
    • Sistemas como RDBMS (bases de datos relacionales), ERP (sistemas de planificación empresarial) o CRM (gestión de relaciones con clientes) que proporcionan los datos necesarios para la respuesta.

    Limitaciones:

    • Cada interacción del usuario requiere que el servidor reenvíe toda la página, lo cual puede ser ineficiente.
    • Hay un retraso perceptible debido a la recarga completa del contenido.

    Modelo Ajax de aplicación web (derecha)

    Flujo de datos:

    • La interfaz del usuario del navegador cliente utiliza un motor Ajax para enviar y recibir datos al servidor sin necesidad de recargar toda la página.

    Componentes del servidor:

    • Similar al modelo clásico, pero aquí el servidor puede devolver datos más específicos y el motor Ajax los procesa dinámicamente en la página.

    Ventajas

    • La experiencia del usuario es más fluida, ya que solo se actualizan partes de la página según sea necesario.
    • Se reduce el uso de ancho de banda, ya que no se necesita recargar todo el contenido.
    • Mejor rendimiento y tiempos de respuesta más rápidos.

    Prototype

    Prototype

    Object():

    • Representa el constructor base de todos los objetos en JavaScript.
    • Todos los objetos en JavaScript descienden de este constructor.

    Object.prototype

    • Es el prototipo predeterminado de los objetos creados a partir del constructor Object.
    • Contiene métodos y propiedades integradas, como .toString() o .hasOwnProperty().
    • Todos los objetos en JavaScript, si no tienen prototipos personalizados, heredan de Object.prototype.

    car:

    • Representa un objeto específico con una propiedad personalizada, este caso, color: 'red'.
    • Este objeto tiene una referencia (__proto__) que apunta a Object.prototype, lo que permite que herede sus métodos y propiedades.

    Relación entre los elementos:

    • El constructor (Object) tiene una propiedad .prototype que apunta a su prototipo (Object.prototype).
    • Cada instancia de objeto tiene una referencia especial, __proto__, que apunta al prototipo del constructor que lo creó.
    • Esto permite que car acceda a métodos y propiedades definidas en Object.prototype.

    Resumen:

    El diagrama nos ilustra cómo JavaScript utiliza prototipos para establecer relaciones entre objetos, implementando un mecanismo de herencia dinámica en el lenguaje. Es fundamental para entender cómo se comportan los objetos en JavaScript y cómo los métodos y propiedades se comparten a través de la cadena prototípica.
